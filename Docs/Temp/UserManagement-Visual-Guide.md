## ?? Badge Styling Details

### Color Scheme
- **Background**: `bg-secondary` (Bootstrap gray)
- **Text**: White (Bootstrap default for secondary badge)
- **Border**: None (Bootstrap badge has no border by default)

### Size & Spacing
- **Padding**: `0.375rem 0.75rem` (matches button height)
- **Font Size**: Inherits from Bootstrap badge class
- **Height**: Matches "Edit Roles" button for alignment

### Interactive States
- **Default**: Gray badge with construction emoji
- **Hover**: Cursor changes to help cursor (?)
- **Click**: Nothing happens (not clickable)
- **Tooltip**: Appears after ~1 second hover

## ?? Color Coding Guide

### Current Implementation (Alpha)
- **Gray Badge**: Feature not yet available
- **Construction Emoji**: Work in progress

### Future Implementation (Production)
- **Red "Disable" Button**: Active user ? Can be disabled
- **Green "Enable" Button**: Disabled user ? Can be re-enabled
- **No Badge**: Feature is fully functional

      title="User disable feature coming in next release"

> "Badge: Disable feature not yet available"

### Keyboard Navigation
- Badge is not focusable (not interactive)
- Users tab past it to next button
- No keyboard trap

- Gray background: `#6c757d`
- White text: `#ffffff`

## ?? Known Quirks

### Bootstrap Badge Behavior
- **Line breaks**: Badge text might wrap on very narrow screens
- **Emoji rendering**: Construction emoji (??) may vary by OS/browser
- **Tooltip delay**: ~1 second default (browser behavior)

### Browser Differences
- **Chrome/Edge**: Emoji renders in color
- **Firefox**: Emoji renders in color
- **Safari**: Emoji may render slightly different
- **IE11**: Not supported (project uses .NET 8)
